## HeadHunterVacancyDataBase

## Описание:

HeadHunter Vacancy Database — консольное приложение на Python для сбора, хранения и анализа данных о вакансиях ведущих компаний России 
на платформе HeadHunter.ru.

Программа:

- Получает актуальные вакансии 10 крупнейших работодателей России (Avito, Точка Банк, Купер, Aston, VK, Контур, 
Тензор, Яндекс, 2ГИС, Skyeng) через HeadHunter API.

- Сохраняет информацию о компаниях и вакансиях в базу данных PostgreSQL.

- Выполняет аналитические запросы через удобное меню CLI.

Проект реализован с использованием ООП-подхода, модульной архитектуры и автоматического обновления данных 
через API HeadHunter.

## Функциональность

### Возможности программы

**После запуска пользователь может выбрать один из запросов:**

- Показать список компаний и количество вакансий

- Показать все вакансии

- Показать среднюю зарплату по вакансиям каждой компании

- Показать вакансии с зарплатой выше средней

- Найти вакансии по ключевому слову

- Выйти из программы

Результаты выводятся в консоль в виде таблицы.

### Проект разделён на несколько слоёв

**1. API слой**

Получает данные с HeadHunter API.

Классы:

- BaseAPISource — базовый класс для работы с API

- HeadHunterVacanciesSource — получение вакансий

- HeadHunterEmployersSource — получение информации о компаниях

- HeadHunterAPI — агрегирует работу источников API

Особенности:

- используется requests.Session

- настроены повторные попытки (Retry)

- вакансии загружаются параллельно через ThreadPoolExecutor

**2. Слой моделей данных**

Используются dataclass-модели:

- Employer

- Vacancy

**3. Слой базы данных**

Работа с PostgreSQL разделена на несколько классов.

- HeadHunterDataBase
    
    - создание базы данных
    
    - создание таблиц
    
    - сохранение данных
    
Создаёт таблицы:

*hh_companies*

| поле           | описание       |
| -------------- | -------------- |
| company_id     | PK             |
| hh_employer_id | id компании HH |
| employer_name  | название       |
| employer_url   | ссылка         |

*hh_vacancies*

| поле           | описание                     |
| -------------- | ---------------------------- |
| vacancy_id     | PK                           |
| hh_vac_id      | id вакансии                  |
| vac_name       | название                     |
| vac_url        | ссылка                       |
| hh_employer_id | id компании                  |
| vac_area       | город                        |
| salary_from    | зарплата от                  |
| salary_to      | зарплата до                  |
| average_salary | вычисляемая средняя зарплата |

- HeadHunterDataBaseManager

Отвечает за SQL-запросы к базе данных.

Методы:

````
get_companies_and_vacancies_count()
get_all_vacancies()
get_avg_salary()
get_vacancies_with_higher_salary()
get_vacancies_with_keyword()
````

- HeadHunterDBCreator

Создаёт и заполняет базу данных:

````
create_and_fill_db()
````

- HeadHunterDBCoordinator

    - выбор нужного SQL запроса
    
    - форматирование результата
    
    - вывод таблицы через tabulate.

Обёртка над менеджером БД.

**4. Координатор приложения**

- HeadHunterDataCoordinator

Связывает:

- API

- создание базы данных

- выполнение запросов

Workflow:

````
HeadHunterAPI → получение данных
HeadHunterDBCreator → создание и заполнение БД
HeadHunterDBCoordinator → выполнение SQL запросов
````

**5. Пользовательский интерфейс**

- UserInteraction

  - вывод приветствия

  - отображение меню

  - получение запроса пользователя

  - ввод ключевого слова

  - запрос на повтор выполнения

Отвечает за взаимодействие с пользователем.

### Логирование

В проекте используется миксин:

- LoggingConfigClassMixin

Позволяет:

- настраивать логгер для любого класса

- писать логи в файл

- выводить логи в консоль

- автоматически создавать папку logs

Формат логов:

````
timestamp - level - logger - module - function:line - message
````

## Установка и запуск

1. Клонировать репозиторий
````
git clone https://github.com/simbarilion/HH_Vacancy_DB.git
cd HH_Vacancy_DB
````
2. Установить зависимости
````
pip install -r requirements.txt

Через poetry:
poetry install
````
3. Создать файл .env для настройки соединения с PostgreSQL
````
DB_USER=your_postgresql_user_name_here
DB_PASSWORD=your_postgresql_password_here
````
4. Запустить программу
````
python main.py
````
